# Deterministic Engine Replacement Forge

## Core doctrine
Build a deterministic study compiler, not an LLM imitation engine.

## Highest law
Never generate user-facing semantic-looking output unless it survives structure, provenance, noise, and distinctness gates.

## What this engine is allowed to do
- extract structured signals
- normalize sources
- classify source origin and trust
- preserve structural hierarchy
- chunk evidence
- reject noise
- generate grounded candidates
- cluster and dedupe candidates
- rank candidates
- build relations
- compile only the fields that are truly supported
- emit provenance for everything visible
- fail closed when evidence is weak

## What this engine must stop pretending to do
- open-ended semantic paraphrasing
- flexible human-like abstraction from arbitrary noisy text
- inventing hooks/traps/distinctions when evidence is thin
- surfacing weak fragments because “something is better than nothing”
- masking uncertainty with polished filler

## Hard quality rules
- Omit over invent
- Provenance over confidence theater
- Structured extraction over prose inference
- Noise rejection over false completeness
- Blank is better than wrong
- A smaller trustworthy engine is better than a larger fake-smart one
- Deterministic means inspectable, benchmarkable, and reproducible

## Mandatory visible truth rules
A visible concept, skill candidate, readiness target, relation, or semantic field must have:
- source origin
- source type
- source evidence
- acceptance score
- reason it passed
- rejection reason if it failed a gate

## Hard rejection rules
Suppress any candidate that is:
- system/browser/LMS chrome
- accessibility fallback text
- JS-disabled warning text
- nav/sidebar/header/footer text
- broken fragment
- truncated phrase
- malformed duplicate
- shell boilerplate
- weak generic week label
- unsupported semantic filler
- suspicious date rendering
- low-evidence semantic role

## Required outputs from this skill
- new isolated engine package
- benchmark harness
- failure corpus
- acceptance matrix
- provenance rules
- migration plan
- regression protection

End of skill.
