# Assignment Readiness Honesty

Use this skill when readiness labels, badges, or assignment-facing nodes risk overstating student preparedness.

## Boundaries

- Readiness must come from prerequisites, evidence, or mastered skills already present in the bundle.
- Weak or partial signals should downgrade to in-progress, blocked, or unavailable states.

## Inputs

- Atlas derivation logic
- Assignment requirement mapping
- Mastery and recovery state

## Outputs

- Honest readiness conditions
- Clear label and copy changes
- Tests for blocked, available, in-progress, earned, and mastered states

## Hard Stops

- Stop if the UI would still imply a claim or unlock action that does not exist.
