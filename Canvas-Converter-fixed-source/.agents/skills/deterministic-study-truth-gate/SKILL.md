# Deterministic Study Truth Gate

## Core doctrine
The system is a deterministic study compiler, not an LLM replacement.

## Highest law
Never show semantic-looking output unless it survives evidence, provenance, and noise-quality gates.

## Non-negotiable rules
- Omit over invent
- Ground over impress
- Structured fields over prose guesses
- Provenance over confidence theater
- Noise rejection over false completeness
- Explicit unknown over fake interpretation
- Stable deterministic output over pseudo-smart output

## Hard block conditions
Suppress or reject any candidate that is:
- obvious site chrome
- accessibility fallback text
- navigation chrome
- JS-disabled text
- boilerplate LMS shell text
- broken phrase fragment
- truncated heading fragment
- malformed duplicate
- pure administrative clutter
- generic week label with no meaningful evidence
- suspicious date output
- unsupported semantic field generated from weak evidence

## Required evidence rules
A visible concept, skill, or readiness item must have:
- a source origin
- a source type
- a source span/snippet or structured field reference
- an evidence score
- a reason it passed the truth gate

## Required suppression rules
Do not surface:
- assignment titles from boilerplate
- concepts from junk headings
- skills from single weak fragments
- due dates from broken parsing
- readiness scores from unsupported mappings
- semantic fields that collapse into restatement

## Required user-facing behavior
When evidence is insufficient:
- show “insufficient evidence”
- show “no trustworthy due date detected”
- show “not enough grounded concept evidence yet”
- show “textbook imported, synthesis gated pending better evidence”
Do not show fake confidence.

## Required outputs from this skill
- noise blocklist
- truth-gate thresholds
- visible provenance
- rejection reasons
- regression fixtures
- exact acceptance criteria

End of skill.
